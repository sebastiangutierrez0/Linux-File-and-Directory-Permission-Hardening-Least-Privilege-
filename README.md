# Linux File and Directory Permission Hardening (Least Privilege)

## Overview
This project focuses on auditing and correcting Linux file and directory permissions to remediate security misconfigurations and reduce the risk of unauthorized access.

## Tech Stack
- Linux
- Bash

## Key Actions
- Audited file and directory permissions
- Removed unnecessary write and execute access
- Applied least-privilege principles to harden the system

## What I Learned
- Interpreting Linux permission and ownership models
- Identifying common permission misconfigurations
- Applying least-privilege controls in a real-world environment
